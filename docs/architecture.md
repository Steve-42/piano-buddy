# 架构说明

## 系统组成

Piano Buddy 是一个纯前端 PWA 应用，由以下部分组成：

### 1. 音频检测模块（audioDetector）
- 通过浏览器麦克风获取声音
- 每 2 秒分析一次频谱，判断是否有钢琴声
- 所有分析在本地完成，音频数据不会存储或上传

### 2. 练习管理模块（usePractice Hook）
- 控制练习的开始/结束
- 根据音频检测结果计算实际弹奏时间
- 5 分钟无琴声自动结束

### 3. AI 鼓励模块（llmService）
- 练习结束后，根据练习数据生成个性化鼓励
- 支持任意 OpenAI 兼容 API（用户自行配置 API Key）
- 未配置 API Key 时使用内置的默认鼓励语

### 4. 数据存储（db）
- 使用浏览器内置的 IndexedDB 数据库（通过 Dexie.js）
- 存储练习记录和用户设置
- 数据完全存储在用户本地

## 数据流动

```
用户弹琴 → 麦克风采集声音 → 频谱分析（本地）→ 判断是否有琴声
                                                    ↓
                                              计时（有声=弹奏时间）
                                                    ↓
                                              练习结束 → 保存到 IndexedDB
                                                    ↓
                                        （可选）调用 LLM API → 生成鼓励语
```

## 页面结构

- **首页**：今日练习进度、AI 提醒、开始按钮
- **练习中**：实时计时、音频状态显示、停止按钮
- **历史记录**：30 天热力图、练习列表、统计数据
- **设置**：每日目标、LLM API 配置
